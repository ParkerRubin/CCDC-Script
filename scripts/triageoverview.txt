triage.ps1

Creates a timestamped triage folder on the target host (C:\IR\TRIAGE\HOST_TIMESTAMP) and exports quick-visibility system, network, and persistence data to text and CSV files for rapid incident response or CCDC triage.

Collected artifacts

system.txt
Host identity and environment: hostname, logged-in user, OS version/build, hardware model, BIOS, domain/workgroup, and IPv4 addresses.

firewall.txt
Windows Firewall profile status (Domain/Private/Public) including whether each profile is enabled and default inbound/outbound behavior.

netstat.txt
Active network connections and listening ports from netstat -ano with associated PIDs.

shares.txt
Configured SMB shares from net share to identify exposed or unexpected shares.

users_admins.txt
Local account visibility: all local users and members of the local Administrators group.

processes.csv
Top running processes by CPU usage with PID, memory usage, and executable path (when available).

services.csv
All Windows services with state, start mode, run account, and service binary path.

scheduled_tasks.csv
Registered scheduled tasks with task path, state, and author for persistence review.

recent_security_events.txt
Recent high-signal Security log events related to logons and account/group changes (e.g., 4624, 4625, 4720–4726, 4732–4733).

summary.txt
Quick snapshot of firewall status, listening TCP ports, local admins, and immediate review notes.
